Implement:
```go
package db

import (
    "database/sql"
    "errors"
    "fmt"
    _ "modernc.org/sqlite"
)

func Open(path string) (*sql.DB, error) {
    if path == "" {
        return nil, errors.New("db: path is required")
    }
    db, err := sql.Open("sqlite", path)
    if err != nil {
        return nil, fmt.Errorf("db: open %q: %w", path, err)
    }
    if err := db.Ping(); err != nil {
        db.Close()
        return nil, fmt.Errorf("db: ping %q: %w", path, err)
    }
    pragmas := []string{
        "PRAGMA journal_mode=WAL",
        "PRAGMA busy_timeout=5000",
        "PRAGMA foreign_keys=ON",
        "PRAGMA synchronous=NORMAL",
        "PRAGMA cache_size=-20000",
        "PRAGMA mmap_size=30000000000",
        "PRAGMA temp_store=MEMORY",
    }
    for _, p := range pragmas {
        if _, err := db.Exec(p); err != nil {
            db.Close()
            return nil, fmt.Errorf("db: %s: %w", p, err)
        }
    }
    // page_size cannot be changed on existing databases; fail silently per D-01
    db.Exec("PRAGMA page_size=4096")
    // D-02: No SetMaxOpenConns — SQLite WAL handles concurrent readers + one writer natively
    // D-03: No SetConnMaxLifetime or SetConnMaxIdleTime — SQLite WAL connections are long-lived
    return db, nil
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> db.Open | User-supplied path reaches sql.Open; path traversal risk |
| db.Open -> SQLite driver | PRAGMA strings are hardcoded; no user input in SQL statements |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering | db.Open path argument | mitigate | Validate path is non-empty before calling sql.Open; Ping failure closes the handle immediately so a misconfigured path doesn't leak a half-open connection |
| T-1-02 | Denial of Service | Shared pool double-close | mitigate | `defer pool.Close()` lives exclusively in main.go; stores never close the shared pool. This plan is the foundation — enforcement comes in Plans 02-05 when Close() is removed from all stores. |
| T-1-03 | Denial of Service | WAL file unchecked growth | accept | SQLite WAL auto-checkpointing runs by default (checkpoint threshold 1000 pages). No manual intervention needed at this phase. Phase 3+ can add periodic PRAGMA wal_checkpoint if profiling shows it. |
</threat_model>
