**internal/scheduler/store.go changes:**

1. Remove the blank driver import: delete `_ "modernc.org/sqlite"` from the import block (line 11). Keep `"database/sql"`.

2. Delete the entire `OpenStore` function (lines 113-131). This removes one of the 6 sql.Open calls.

3. Delete the entire `Close` method (lines 143-148). The scheduler.Close() has NO ownership guard — it unconditionally calls `s.db.Close()`. This was the most dangerous pattern because calling it on a shared pool would close the connection while other stores still hold references.

5. Update the struct doc comment (lines 106-108) to reflect that Store no longer exposes DB() or Close().

6. Keep `NewStoreWithDB` exactly as-is (lines 135-141) — it's already correct: no owned field, no side effects on the pool.

**CRITICAL:** After removing these functions, packages that import scheduler and call `schedStore.Close()`, `schedStore.DB()`, or `scheduler.OpenStore()` will fail to compile. This is EXPECTED — Plans 05 and 06 update all callers. The `go build ./internal/scheduler/` must still pass (it doesn't depend on external callers). The scheduler's own tests must pass with `go test ./internal/scheduler/ -count=1`, which is handled in Tasks 2 and 3.
