## Codebase Patterns
- Use `slog` for structured logging throughout (not zap yet, will migrate later)
- Config uses manual env parsing (no envconfig dep yet) — `internal/config/env.go` has helpers
- Telegram user IDs are strings (converted from int64 via strconv.FormatInt)
- Conversation handling launched in goroutines via `go b.handleConversation(c)`
- Wiki store uses `go-git/go-git/v5` for Git operations (not shell commands)
- Wiki page YAML must include `schema_version: 1` and `prompt_version: vN|ingest_vN`
- Slug function collapses consecutive hyphens in wiki page filenames
- chromem-go uses `Similarity` (not Distance) on Result — range [-1, 1]
- chromem-go embedding: use `NewEmbeddingFuncOpenAICompat` for OpenAI-compatible endpoints
- Budget tracking: `budget.Tracker` in bot — check `CanAfford()` and `IsHardBudgetExceeded()` before LLM calls, `RecordUsage()` after
- Context enforcement: `EnforceLimit()` replaces manual `ShouldSummarize()`+`Summarize()` — 3-tier: summarize → trim → truncate
- `/status` Telegram command for budget and context visibility
